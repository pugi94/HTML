01

프로젝트 목표

BMS 그룹웨어가 달성하고자 하는 5가지 핵심 목표

🔗

업무 도구 통합 — 원스톱 플랫폼

현재 분산된 업무 도구(카카오톡, 이메일, 개별 일정 관리, 파일 공유 등)를 하나의 플랫폼으로 통합합니다. 사용자는 BMS 하나로 커뮤니케이션, 업무 관리, 일정, 파일을 모두 처리할 수 있습니다.

⚡

업무 자동화 / 효율화

보고서 생성, 블로그 포스팅, 데이터 취합 등 반복적인 업무를 자동화하여 시간을 절감합니다. 슬랙 → 시트 → 다우 연동 파이프라인을 통해 수작업을 최소화합니다.

📊

데이터 중앙화 — 종합리포트.com

구글 스프레드시트, GA 분석 데이터, 각 앱의 산출물을 종합리포트.com으로 집약합니다. 흩어진 데이터를 한 곳에서 조회·분석하여 의사결정 속도를 높입니다.

🛡️

외부 서비스 의존도 축소

유료 SaaS 도구에 대한 의존을 줄이고, 자체 개발 시스템으로 데이터 주권을 확보합니다. 비용 절감과 함께 사내 요구에 맞는 커스터마이징이 가능해집니다.

🧠

데이터 온톨로지 구축 — 연동·수기 입력 자동화

시스템 간 데이터 연동과 수기 입력을 자동화하여 맞춤형 업무 체계를 구축합니다. 온톨로지(Ontology) 개념을 적용해 데이터 간의 관계와 의미를 구조화하고, 각 시스템(슬랙, 시트, 다우, GA 등)에서 생성되는 데이터가 자동으로 분류·연결·축적되는 지식 기반 시스템을 지향합니다. 궁극적으로 수작업 데이터 입력을 제로에 가깝게 줄이고, 데이터가 스스로 맥락을 갖는 자동화 파이프라인을 완성합니다.

프로젝트 비전

모든 업무 데이터가 한 곳에 모이고, 반복 업무는 자동화되며,
50인 조직에 꼭 맞는 경량 그룹웨어로 일하는 방식을 바꾼다.

02

현 그룹디 문제점

BMS 그룹웨어 구축이 필요한 배경이 되는 현재 조직의 핵심 문제

🔴 업무 환경

업무 도구 분산 심각

카카오톡, 이메일, 구글 시트, 다우오피스 등 업무 도구가 각각 분리되어 있어 정보 확인을 위해 여러 앱을 오가야 하며, 맥락이 단절됩니다.

수기 입력·반복 작업 과다 심각

동일 데이터를 시트, 다우, 보고서 등에 수동으로 중복 입력하는 작업이 빈번하여 시간 낭비와 입력 오류가 발생합니다.

커뮤니케이션 누락·지연 심각

카카오톡 중심의 커뮤니케이션으로 업무 지시·공유가 대화 속에 묻히고, 읽지 못하거나 지연 전달되는 경우가 잦습니다.

데이터 흩어짐 심각

파일과 데이터가 개인 PC, 구글 드라이브, 시트, 이메일 첨부, 사내 NAS 등에 산재되어 있고, PDF·PPTX·Excel 등 다양한 포맷으로 흩어져 있어 필요한 자료를 찾는 데 시간이 과도하게 소요됩니다.

🟡 데이터 / 시스템

데이터 버전 관리 부재 주의

스프레드시트 기반 작업에서 최신본과 구버전이 혼재되어 잘못된 데이터로 작업하는 경우가 발생합니다.

다우오피스 연동 제약 주의

다우오피스 API의 기능 제한으로 슬랙 등 외부 시스템과의 자동 연동이 어렵고, 현재 연동 작업이 중단된 상태입니다.

외부 SaaS 비용 부담

여러 외부 서비스에 분산 과금되어 비용 관리가 어렵고, 조직 규모 대비 과도한 라이선스 비용이 발생합니다.

🟣 조직 / 업무 프로세스

업무 현황 파악 어려움

누가 어떤 업무를 진행 중인지 실시간으로 확인할 수 없어 업무 중복이나 공백이 발생합니다.

보고 체계 비효율

각종 보고 데이터를 수동으로 취합해야 하며, 보고서 작성에 실제 업무 시간 대비 과도한 시간이 소요됩니다.

경영진 보고 비효율 심각

대표에게 업무 현황을 보고하기 위해 각 팀 데이터를 수동으로 취합·가공해야 하며, 실시간 경영 현황을 한눈에 파악할 수 있는 도구가 없어 보고 자체가 추가 업무가 되고 있습니다.

신규 입사자 온보딩 어려움

업무 매뉴얼, 히스토리, 자료가 체계적으로 정리되어 있지 않아 신규 입사자의 적응 기간이 길어집니다.

의사결정 속도 느림

필요한 데이터에 빠르게 접근하기 어려워 의사결정이 지연되고, 실시간 현황 기반의 빠른 판단이 불가능합니다.

💡

핵심 요약 — 위 문제들은 결국 "도구 분산 → 데이터 분산 → 수작업 증가 → 비효율 누적"이라는 하나의 악순환 구조로 연결됩니다. BMS 그룹웨어는 이 순환을 끊고, 통합 플랫폼 위에서 데이터가 자동으로 흐르는 구조를 만드는 것이 핵심입니다.

PM (Project Manager) 역할

🎯 PM — 프로젝트 매니저 핵심 역할

현재 마케팅팀과 전략기획팀이 PM 역할을 겸임하고 있으며, 1명의 PM이 여러 거래처를 동시에 담당하는 구조입니다. 프로젝트의 시작부터 완료까지 전체 흐름을 관리하는 허브 역할을 수행합니다.

  • 클라이언트 요청 접수 및 배분 — 클라이언트의 요청을 최초 접수하고, 업무 성격에 따라 적절한 팀(디자인/콘텐츠/영상 등)에 배분
  • 프로젝트 일정·마감 관리 — 프로젝트별 타임라인 설정, 마일스톤 추적, 납기 준수 관리
  • 팀간 업무 조율 및 인수인계 — 기획→디자인→콘텐츠→영상 등 단계별 전환 시 맥락 전달 및 진행 상황 조율
  • 클라이언트 보고 및 커뮤니케이션 — 진행 상황 공유, 성과 리포트 작성, 피드백 반영 관리
⚠️

현재 PM 운영의 한계 — 1인 다수 거래처 담당으로 과부하 발생, 전용 도구 없이 카톡·시트·메일로 프로젝트를 관리하여 현황 파악이 어렵고, PM 간 업무 인수인계 시 히스토리 손실이 빈번합니다. BMS 칸반보드와 대시보드가 이 문제를 해결하는 핵심 기능이 됩니다.

팀 구성 및 팀별 병목

📈

마케팅팀

클라이언트 대응, 광고 캠페인 기획·운영, 매체 관리. 리테이너 및 단건 클라이언트의 일상 운영을 담당합니다.

⚠ 병목: 클라이언트 커뮤니케이션 채널 분산 (카톡/이메일/전화 혼재), 성과 데이터 수동 취합
💡

전략기획팀

마케팅 상품 개발, 사내 시스템 기획(BMS 포함), 회사 내부 프로세스 확립. 신규 서비스·상품 설계 및 조직 업무 체계를 구축합니다.

⚠ 병목: 팀간 인수인계 지연 (기획 → 디자인/콘텐츠 전달 시 맥락 손실), 내부 프로세스 문서화 부재
🎨

디자인팀

광고 소재 디자인, 랜딩페이지, 브랜딩 비주얼 제작. 기획팀·콘텐츠팀과 긴밀히 협업합니다.

⚠ 병목: 기획 → 디자인 인수인계 시 요구사항 불명확, 수정 반복
✏️

콘텐츠팀 (블로그)

블로그 포스팅, 카피라이팅, SEO/AEO 콘텐츠 제작. 마케팅 성과에 직접 연결되는 콘텐츠를 생산합니다.

⚠ 병목: 콘텐츠 발행 후 성과 데이터(GA) 취합이 수동, 리포트 작성 시간 과다
🎬

영상팀

영상 광고 소재, 숏폼/롱폼 콘텐츠 기획 및 제작. 대용량 파일 작업이 잦습니다.

⚠ 병목: 대용량 파일 공유 어려움, 영상 피드백·수정 이력 관리 부재
💰

재무팀

매출/매입 관리, 클라이언트 계약·정산, 월간 재무 보고. 다우오피스와 스프레드시트를 주로 사용합니다.

⚠ 병목: 계약·정산 데이터 수기 입력, 시트↔다우 간 이중 작업

계약 구조

클라이언트와의 계약은 아래 세 가지 형태가 혼재되어 있으며, 각 형태에 따라 업무 주기와 보고 방식이 달라집니다.

🔄

월 정기 계약 (리테이너)

월 단위 정기 운영, 월간 리포트 필수, 상시 커뮤니케이션

📌

프로젝트 단건 계약

기간 한정 프로젝트, 마일스톤별 산출물 납품, 완료 후 종료

🔀

혼합형 (정기 + 단건)

PM 작업 구조 × BMS 시스템 설계

그룹디 PM의 6단계 업무 흐름과 각 단계를 지원하는 BMS 기능 매핑

분석 → 기획 → 제안 → 제작 → 실행 → 보고서
01
분석 Analysis
예전 방식 (AS-IS)
  • 플레이스 광고·GFA 광고 성과 데이터를 수동으로 스프레드시트에 복사
  • 클라이언트 요청을 카톡·이메일에서 개별 취합
  • 경쟁사 리서치·플레이스 현황 자료를 개인 PC·NAS에 저장
  • 기존 캠페인 성과를 다우 Works·시트에서 수동 검색
BMS 지원 기능 (TO-BE)
  • GA/AEO·광고 매체 데이터 자동 연동 → 경영진 대시보드
  • 클라이언트 요청 접수 → 칸반보드 자동 카드 생성
  • 리서치 자료 → 파일 드라이브 중앙 저장 (NAS 대체)
  • 캠페인 성과 이력 → 종합리포트.com 자동 축적
02
기획 Planning
예전 방식 (AS-IS)
  • 플레이스·GFA 캠페인 전략을 개별 문서로 작성 (포맷 비통일)
  • 콘텐츠/매체 기획을 시트·다우 Works로 관리
  • 클라이언트 요청사항이 카톡 대화에 묻힘
  • 일정·예산 설계를 별도 시트에 수기 입력
BMS 지원 기능 (TO-BE)
  • 프로젝트 보드 생성 → 전략·일정·예산 통합 관리
  • 칸반 카드에 기획 문서·매체 플랜 첨부
  • 클라이언트 요청사항 → 채팅·칸반 연동 추적
  • 캘린더 연동으로 마일스톤·마감일 자동 반영
03
제안 Proposal
예전 방식 (AS-IS)
  • 제안서(PPT) 작성 후 이메일·카톡으로 공유
  • 클라이언트 피드백이 여러 채널에 분산
  • 수정 버전 관리 안 됨 (최종_최종_진짜최종.pptx)
  • 프레젠테이션 일정 조율을 카톡으로 진행
BMS 지원 기능 (TO-BE)
  • 제안서 → 파일 드라이브 버전 관리 (자동 이력)
  • 피드백 → 칸반 카드 코멘트로 일원화
  • 수정 내역 → 파일 버전 비교 및 최신본 자동 표시
  • 프레젠테이션 일정 → 캘린더 초대 및 알림
04
제작 Production
예전 방식 (AS-IS)
  • 기획→디자인 시안물·영상 콘텐츠·플레이스 관리 인수인계 시 맥락 손실
  • 디자인 시안·영상 피드백이 카톡·이메일에 분산
  • 영상 콘텐츠 대용량 파일 공유 어려움 (사내 NAS·메일 한계)
  • 각 팀(디자인/콘텐츠/영상) 제작 진행 상황을 PM이 개별 확인
BMS 지원 기능 (TO-BE)
  • 칸반보드 드래그앤드롭으로 팀간 업무 이관 (맥락 유지)
  • 카드 내 코멘트·멘션으로 피드백 집중
  • 파일 드라이브 대용량 업로드 + 미리보기
  • 보드 뷰에서 전체 제작 현황 실시간 파악
05
실행 Execution
예전 방식 (AS-IS)
  • 플레이스·GFA 광고 집행 후 성과 데이터를 매체별 수동 수집
  • 거래처에 제출할 마케팅 플래너(일정·전략 문서)가 없음
  • 집행 현황을 시트·다우 Works에 수기 기록
  • 이슈 발생 시 카톡으로 보고 → 히스토리 유실
  • 예산 소진율을 별도로 계산, 거래처 계약서 열람 불가
BMS 지원 기능 (TO-BE)
  • 실행 데이터 → 대시보드 자동 집계
  • 거래처용 마케팅 플래너 자동 생성 (일정·전략·예산 통합)
  • 칸반 카드 상태를 '실행 중'으로 자동 전환
  • 이슈 → 채팅 + 칸반 연동으로 기록 보존
  • 거래처 계약 정보 열람 → 권한별 계약서 조회
  • 예산/성과 데이터 → 경영진 대시보드 실시간 반영
06
보고서 Report
예전 방식 (AS-IS)
  • GA·플레이스·GFA 매체 데이터를 수동으로 취합
  • 보고서를 PPT·Excel로 수작업 제작, 각자 따로 나감 (통일 양식 없음)
  • 경영진 보고를 위해 다시 별도 요약 작업
  • 거래처 계약서 열람 불가, 과거 보고서 검색·재활용 어려움
BMS 지원 기능 (TO-BE)
  • 보고서 앱 연동 → 데이터 자동 취합·시각화
  • 경영진 대시보드에서 실시간 리포트 조회
  • 종합리포트.com으로 클라이언트별 보고서 자동 생성
  • 보고서 아카이브 → 파일 드라이브 자동 저장·검색
설계 핵심 원칙

PM의 6단계 워크플로우 전체를 BMS 안에서 끊김 없이 처리하고,
각 단계에서 생성된 데이터가 다음 단계로 자동 연결되는 구조를 만든다.

기능 요구사항

MVP 핵심 기능과 2단계 확장 기능

Phase 1 — MVP 3개월 내 구현 대상 핵심 기능

🔐 로그인 / 권한 관리 P0

  • 사번 또는 이메일 기반 로그인
  • 역할 기반 접근 제어 (RBAC): 관리자, 팀장, 일반 사원
  • 비밀번호 재설정 및 세션 관리
  • 신규 사용자 등록은 관리자 승인 방식

📊 대시보드 (메인 화면) P0

  • 로그인 후 첫 화면으로 표시
  • 오늘의 일정, 나에게 할당된 업무, 최근 알림 요약
  • 팀별 공지사항 위젯
  • 빠른 액세스 메뉴 (채팅, 칸반, 캘린더, 파일)

👔 경영진 보고 대시보드 P0

  • 대표/경영진 전용 대시보드 뷰 (별도 권한)
  • 전사 프로젝트 현황 한눈에 조회 (진행 중/완료/지연)
  • 팀별 업무 부하 및 성과 요약 (자동 취합)
  • 클라이언트별 계약 현황·매출 데이터 연동
  • GA/AEO 성과 데이터 자동 연동 및 시각화
  • 월간/주간 리포트 자동 생성 (수동 취합 제거)
  • 보고서 앱·스프레드시트·종합리포트.com 데이터 통합 표시

  • 1:1 다이렉트 메시지
  • 그룹 채팅방 생성 및 관리

  • 읽음 확인 및 알림 연동
  • 채팅 내 메시지 검색

📋 칸반보드 (업무 관리) P1

  • 프로젝트 단위 보드 생성
  • 컬럼 커스터마이징 (예: To Do → In Progress → Review → Done)

  • 드래그 앤 드롭으로 상태 변경

  • 알림 유형: 업무 할당, 일정 리마인더, 채팅 멘션, 파일 공유

리스크 및 고려사항

기술, 조직, 운영 측면의 주요 리스크

🔧 기술 리스크

WebSocket 기반 채팅의 동시 접속 처리 및 재연결 로직이 필요합니다. 파일 스토리지 사용량 증가 시 비용 관리 방안을 수립해야 하며, 사내 민감 데이터에 대한 암호화 및 접근 로그 관리가 필수적입니다.

👥 조직 리스크

3개월 내 MVP 완성을 위해 최소 프론트엔드 1명 + 백엔드 1명 확보가 필요합니다. 개발 중 요구사항 변경을 최소화하기 위한 스프린트 단위 관리를 권장하며, 기존 도구에서의 전환 시 사용자 교육 및 온보딩 계획이 필요합니다.

⚙️ 운영 리스크

자체 서버 vs 클라우드(AWS/GCP) 인프라 결정이 필요하고, 데이터 백업 주기 및 재해 복구 계획을 수립해야 합니다. 출시 후 지속 운영을 위한 전담 담당자 지정이 요구됩니다.
